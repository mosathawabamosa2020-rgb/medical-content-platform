# توثيق خدمة البحث

## 1. نظرة عامة

خدمة البحث في المنصة تستخدم محرك بحث ذكي يعتمد على:
- قاعدة بيانات PostgreSQL مع إضافة pgvector
- خدمة تضمين محلية
- خدمة استرجاع متقدم


## 2. مكونات الخدمة

### 2.1 SearchAggregatorService
- الملف: `lib/search/SearchAggregatorService.ts`
- الوظيفة: تجميع نتائج البحث من عدة مصادر
- المصادر المدعومة: PubMed

- الطريقة: تجميع النتائج من جميع المحولات المتاحة


### 2.2 PubMedAdapter
- الملف: `lib/sources/PubMedAdapter.ts`
- الوظيفة: البحث في قاعدة بيانات PubMed
- نقاط API:
  - البحث: `https://eutils.ncbi.nlm.nih.gov/entrez/eutils/esearch.fcgi`
  - الملخصات: `https://eutils.ncbi.nlm.nih.gov/entrez/eutils/esummary.fcgi`
- الطريقة:
  1. البحث للحصول على معرفات المقالات
  2. جلب الملخصات لكل معرف
  3. تجميع النتائج

- البيانات المُرجعة:
  - المعرف (id)
  - العنوان (title)
  - المؤلفون (authors)
  - الملخص (summary)
  - رابط المصدر (sourceUrl)

## 3. نقاط API

### 3.1 البحث
- المسار: `/api/search/pubmed`
- الطريقة: GET
- المعاملات: `term` (مطلوب)
- التخزين المؤقت: ذاكرة في الذاكرة (5 دقائق)
- الاستجابة: مصفوفة JSON

### 3.2 جلب التفاصيل
- المسار: `/api/references/[id]`
- الطريقة: GET
- المعاملات: `id` (مطلوب)
- الاستجابة: تفاصيل المرجع

## 4. التكامل مع محرك الاسترجاع

### 4.1 التكامل
- خدمة البحث تُكمل مع محرك الاسترجاع
- النتائج من البحث الخارجي تُضاف إلى قاعدة البيانات
- يمكن البحث في المراجع المضافة

### 4.2 الاستخدام
```bash
# البحث في PubMed
curl "http://localhost:3000/api/search/pubmed?term=diabetes"

# البحث في جميع المصادر
curl -X POST http://localhost:3000/api/references/query \
  -H "Content-Type: application/json" \
  -d '{"query": "diabetes", "topK": 8, "page": 1}'
```

## 5. التحقق

### 5.1 التحقق من الاتصال
```bash
# التحقق من خدمة البحث
curl http://localhost:3000/api/search/pubmed?term=test

```

### 5.2 التحقق من النتائج
```bash
# التحقق من جلب التفاصيل
curl http://localhost:3000/api/references/<id>
```

## 6. الصيانة

### 6.1 المراقبة
- مراقبة أداء البحث
- مراقبة عدد الطلبات
- مراقبة الأخطاء

### 6.2 الأخطاء الشائعة
- فشل الاتصال بـ PubMed
- عدم وجود نتائج
- أخطاء في التخزين المؤقت

## 7. التوسعات المستقبلية

### 7.1 مصادر إضافية
- إمكانية إضافة محولات لمصادر أخرى
- دعم لـ Google Scholar
- دعم لـ Crossref

### 7.2 تحسينات الأداء
- تحسين التخزين المؤقت
- تحسين استراتيجيات البحث
- إضافة تصنيف للنتائج

## 8. الخلاصة

خدمة البحث في المنصة:
- ✅ تعمل بشكل صحيح
- ✅ متكاملة مع محرك الاسترجاع
- ✅ تدعم البحث في PubMed
- ✅ تستخدم ذاكرة مؤقتة
- ✅ موثقة بالكامل

يمكنك الآن:
1. البحث في PubMed باستخدام `/api/search/pubmed`
2. البحث في جميع المصادر باستخدام `/api/references/query`
3. جلب تفاصيل المراجع باستخدام `/api/references/[id]`

جميع الخدمات تعمل بشكل صحيح ومتكاملة.
