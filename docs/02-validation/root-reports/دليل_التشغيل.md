# دليل التشغيل الكامل - منصة المحتوى الطبي

## 1. المتطلبات الأساسية

### 1.1 قاعدة البيانات
- PostgreSQL مع إضافة pgvector
- إعداد المستخدم postgres وكلمة المرور

### 1.2 Node.js
- الإصدار: v18.16.0 أو أحدث

- npm أو yarn لإدارة الحزم

### 1.3 المتغيرات البيئية
- DATABASE_URL
- OPENAI_API_KEY (اختياري للوضع المستقل)
- SYSTEM_MODE (online/offline)
- REDIS_URL (اختياري)

## 2. خطوات التشغيل

### 2.1 إعداد البيئة
```bash
# نسخ ملف المتغيرات
cp .env.example .env

# تثبيت التبعيات
npm install
```

### 2.2 تشغيل قاعدة البيانات
```bash
# باستخدام Docker Compose
docker-compose up -d


# أو تشغيل PostgreSQL محلي
# تأكد من تثبيت pgvector
```

### 2.3 تطبيق الترحيلات
```bash
# تطبيق الترحيلات
npx prisma migrate deploy


# زرع البيانات الأولية
npx prisma db seed
```

### 2.4 بناء التطبيق
```bash
# بناء التطبيق
npm run build


# تشغيل التطبيق في وضع التطوير
npm run dev


# تشغيل التطبيق في وضع الإنتاج
npm start
```

## 3. العمليات الأساسية

### 3.1 المصادقة
```bash
# تسجيل الدخول
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email": "admin@example.com", "password": "admin123"}'
```

### 3.2 رفع المراجع
```bash
# رفع ملف PDF
curl -X POST http://localhost:3000/api/admin/ingestion \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{"deviceId": "<device-id>", "title": "Test Reference", "filePath": "/uploads/test.pdf"}'
```

### 3.3 التحقق من المراجع
```bash
# الموافقة على المرجع
curl -X POST http://localhost:3000/api/admin/verification/approve \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{"referenceId": "<ref-id>", "reviewerId": "<user-id>", "comment": "Approved for use"}'
```

### 3.4 البحث
```bash
# البحث في المحتوى
curl -X GET "http://localhost:3000/api/search?query=test%20device"
```

### 3.5 توليد المحتوى
```bash
# توليد محتوى جديد
curl -X POST http://localhost:3000/api/content/generate \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{"topic": "Test Device", "tone": "professional", "platform": "web"}'
```

## 9. النسخ الاحتياطي والاستعادة

### 9.1 النسخ الاحتياطي
```bash
# نسخ احتياطي كامل
docker-compose exec db pg_dump -Fc -f backup_$(date +%Y%m%d_%H%M%S).dump medical

```

### 9.2 الاستعادة
```bash
# استعادة النسخة الاحتياطي
docker-compose exec -T db pg_restore -d medical -Fc -f backup_YYYYMMDD_HHMMSS.dump
```
